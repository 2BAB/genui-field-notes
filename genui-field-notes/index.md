# Introduction

A polished card appearing inside a chat message, a model generating JSON for a renderer, or a page reorganizing its layout and theme based on user input: today, all of these may be placed under the same term, Generative User Interface, aka GenUI.

For engineers, it is easier to narrow the discussion to the technical layer: how client-side UI dynamically generates, adapts, or assembles interfaces from intent, state, tools, data, and feedback. From this angle, GenUI covers not only the Web pages and apps we use every day, but also experiments that try to generate game interfaces or game experiences in real time.

This minibook will move through several early practices that are easier to observe: OpenUI/Thesys C1, Google A2UI, Vercel's practice, and the concrete questions around the expression layer, runtime, and agent development. Many judgments here are still stage observations. We care more about which differences can already be seen and which questions are worth further testing. We will not try to cover everything; Coding Agents generating product prototypes and games are outside the scope of this book for now.
